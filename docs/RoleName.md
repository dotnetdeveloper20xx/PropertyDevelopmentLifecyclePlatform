## How to Demo

1. **Drop the database** (delete the .db file or use SQL Server Management Studio)
2. **Start the backend**: `dotnet run` from `backend/src/BuildEstate.API`
3. The seeders automatically create: roles → permissions → demo users → demo data
4. **Start the frontend**: `ng serve` from `frontend/`
5. Navigate to `http://localhost:4200`
6. Login as any user from the table above
7. Each user will see ONLY their permitted sidebar items
8. Each user can perform ONLY the tasks listed for their role

---

## Known Behaviour

- If a user manually types a URL they don't have access to, the API returns 403 Forbidden
- The sidebar only shows menu items relevant to the logged-in role
- All actions are audit-logged (visible in Admin → Audit Log)
- Activity feed on each module page shows recent changes
- CSV export available on all list pages
- Toast notifications confirm all create/edit operations
